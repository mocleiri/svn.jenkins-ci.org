Contrôle comment Hudson autorise les lancements par JNLP.
Lancement uniquement à partir des "Détails de l'ordinateur". Un login est demandé si la sécurité est activée.
Ceci est le comportement par défaut. Dans ce mode, Hudson fournit le lien de lancement JNLP sur la vue de l'ordinateur. Si la sécurité est activée, le lien ne sera disponible que pour les utilisateurs identifiés.
Lancement à partir de la page d'accueil. Login demandé si la sécurité est activée.
Dans ce mode, Hudson fournit le lien de lancement JNLP à la fois sur la vue de l'ordinateur et sur le panneau du Lanceur de build sur le côté de la fenêtre. Si la sécurité est activée, alors le lien ne sera disponible que pour les utilisateurs identifiés.
Lancement seulement à partir des "Détails de l'ordinateur". Un login n'est jamais demandé.
Dans ce mode, Hudson fournit le lien de lancement JNLP sur la vue de l'ordinateur. Le lien sera toujours disponible quand la machine esclave est déconnectée.
Attention! Dans ce model, la sécurité est désactivée. Les esclaves peuvent donc être lancés par toutes les personnes qui ont accès au serveur. Les esclaves ont la capacité d'exécuter n'importe quel code sur la machine maître. Ne sélectionnez pas cette option si vous n'avez pas bien conscience des risques encourus.
