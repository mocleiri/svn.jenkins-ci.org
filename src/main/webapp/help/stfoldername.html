The list of folder to be checked out to the root of the workspace.

Example 1 - Single project

MyProjects/alpha

Example 2 - Single project with multiple source folders checked out on top of each other

MyProjects/alpha
MyProjects/beta
MyProjects/delta

Example 3 - Single project with multiple source folders checked out to different locations

Provide a comma seperated list of folders. The first column is the starteam folder. The second column is the checkout destination relative to the hudson workspace. 

MyProjects/alpha
MyProjects/beta/src,    src/main/java
MyProjects/delta/src,   src/test/java
MyProjects/tests/data,  src/test/resources
Note: It is recommended that projects adopt a proper structure. This feature assists difficult project structures that should be transitioned to more a maintainable structure.

Why

O horrible world... why .. o why?

This helps when migrating badly structured projects to maven projects. For mediaum to large projects (40+ projects, 10,000+ files, 5+ destination environments, lots of circular dependencies, tc) it is difficult to migrate from an existing "working" build system (typically held together by ANT scripts) to maven in a single leap and flexibility is needed. This provides that flexibility.
