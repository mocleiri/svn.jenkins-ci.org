Specifies the execution schedule in a cron notation when a differential backup should be done. Look at Wikipedia for more information.

A differential backup stores only complete files whose modification is done after the last full backup. To restore a configuration state on a differential backup you need to restore the full backup first and then the differential backup you need.

Note: For a differential backup at least one full backup is needed. If no full backup is available it will be done instead of the first differential backup.
