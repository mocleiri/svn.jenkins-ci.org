Problem creating ~/.m2

Maven needs to have the ~/.m2 directory for it to operate correctly. This directory will store the maven local repository, as well as configuration files like ~/.m2/settings.xml. Because of this, Hudson tries to create this directory before it invokes Maven, and if it fails, it report an error like this: 

Failed to create /var/www/.m2

In some environment, the user account that runs container may not have a permission to create such a directory. For example, maybe hudson is running as nobody with home directory set to /var/www. If so, please create this directory manually.

See the relevant e-mail thread and a bug report.
