getEnvVars

public java.util.Map<java.lang.String,java.lang.String> getEnvVars()
Description copied from class: Run
Returns the map that contains environmental variable overrides for this build.

BuildSteps that invoke external processes should use this. This allows BuildWrappers and other project configurations (such as JDK selection) to take effect.

On Windows systems, environment variables are case-preserving but comparison/query is case insensitive (IOW, you can set 'Path' to something and you get the same value by doing '%PATH%'.) So to implement this semantics the map returned from here is a TreeMap with a special comparator.
