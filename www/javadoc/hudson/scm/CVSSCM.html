CVS.

I couldn't call this class "CVS" because that would cause the view folder name to collide with CVS control files.

This object gets shipped to the remote machine to perform some of the work, so it implements Serializable.

getModuleRoots

public FilePath[] getModuleRoots(FilePath workspace)
Description copied from class: SCM
Gets the top directories of all the checked out modules.

Some SCMs support checking out multiple modules inside a workspace, which creates directory layout like this: 

 workspace  <- workspace root
  +- xyz    <- directory checked out by SCM
      +- .svn
      +- build.xml  <- user file
  +- abc    <- second module from different SCM root
      +- .svn
      +- build.xml  <- user file
This method takes the workspace root as a parameter, and is expected to return all the module roots that were checked out from SCM.

For normal SCMs, the array will be of length 1 and it's contents will be identical to calling SCM.getModuleRoot(FilePath).

Overrides:
getModuleRoots in class SCM
Parameters:
workspace - The workspace root directory
Returns:
An array of all module roots.
