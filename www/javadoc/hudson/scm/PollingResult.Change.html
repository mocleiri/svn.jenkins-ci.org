INCOMPARABLE

public static final PollingResult.Change INCOMPARABLE
The state as of baseline is so different from the current state that they are incomparable (for example, the workspace and the remote repository points to two unrelated repositories because the configuration has changed.) This forces Hudson to schedule a build right away.

This is primarily useful in SCM implementations that require a workspace to be able to perform a polling. SCMs that can always compare remote revisions regardless of the local state should do so, and never return this constant, to let Hudson maintain the quiet period behavior all the time.

This constant is not to be confused with the errors encountered during polling, which should result in an exception and eventual retry by Hudson.
